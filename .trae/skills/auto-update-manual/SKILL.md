# 自动更新操作手册

这个 Skill 帮助你根据代码变动自动更新项目的操作手册。

## 使用时机
- 当用户修改了代码并影响了用户可见的功能时。
- 当你需要确保文档与最新的代码实现保持同步时。
- **注意**：如果是纯粹的 bug 修复（不改变功能逻辑或界面交互），不需要更新手册。

## 执行步骤
1.  **分析变更**:
    - 仔细查看最近的代码修改，理解哪些功能发生了变化（新增、修改或删除）。
    - 确定这些变化是否影响用户操作或界面展示。
    - **排除 Bug 修复**: 如果变更仅是为了修复代码错误，且未引入新功能或改变现有操作流程，则跳过更新手册。

2.  **定位手册**:
    - 在 `manual/` 目录下查找与变更功能相关的 Markdown 文件。
    - 使用 `LS` 或 `Glob` 工具来搜索文件名。

3.  **读取手册**:
    - 读取目标手册文件的当前内容，了解现有的文档结构和描述风格。

4.  **更新内容**:
    - **新增功能**: 在合适的位置添加新功能的描述和操作说明。
    - **修改功能**: 更新现有描述以反映最新的行为。
    - **删除功能**: 移除已废弃功能的说明。
    - **保持一致性**: 确保新增或修改的内容与现有文档的语言风格（中文）和格式保持一致。
    - **图片说明**: 如果变动涉及界面变化，虽然你不能截图，但请在文档中用文字描述应该展示的截图内容（例如：`![新功能截图](./images/new-feature.png)`），并提示用户后续补充截图。

5.  **验证**:
    - 检查更新后的文档是否通顺、准确。
    - 如果变动较大，可以请求用户确认。
