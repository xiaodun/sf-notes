在 sf-notes 里集中管理常用接口：按环境分组、记住 URL 与参数、一键发送并查看响应，适合日常联调与复现问题。

# 能做什么

- **分组与环境**：为接口建分组；每个分组可配置多个「前缀」（如 dev / prod 的 base URL），每次只启用一个，避免改来改去。
- **公共 Headers**：分组级配置，该分组下所有请求都会自动带上（如鉴权、区域等）。
- **保存 API**：记录名称、方法、路径；路径可与当前选中的前缀拼成完整地址；自动记住上次的 Query 或 Body。
- **发送与结果**：点击发送即发起真实请求；状态码与响应体会展示在右侧面板；再次打开同一条 API 会恢复上次请求与响应。
- **大响应**：响应体很大时仍可浏览：公共结构会优先展示并支持折叠与搜索；特别大的列表等可按需在树里继续展开；需要全文定位时可用搜索（由服务端扫描，不必整包下载到浏览器）。
- **导出**：在「发送」旁点 **导出**，从服务端缓存拉取**上游接口返回的响应体**（合法 JSON 存为 `.json` 且仅含解析后的数据；非 JSON 为 `.txt`）。超大正文超过约 48MB 时改为流式下载 UTF-8 原文 `.txt`。

# 怎么用

1. 在首页进入 **API**。
2. **新建分组**（若还没有）：填分组名称；按需添加多个前缀，用单选框选定当前要用的环境；按需填写公共 Headers。
3. **新建 API** 或选中已有条目：填写名称、方法；若已配置分组前缀，在右侧「前缀 + 路径」一体框里选好环境，再填路径（或仍可直接填完整 `http(s)://` 地址）。
4. GET 时在 Query 区写参数；POST 时在 Body 区写 JSON，可用「格式化」整理缩进。
5. 点 **发送**，在下方查看状态与响应体；树形区域可展开折叠，搜索框支持本地与远程配合查找。
6. 发送旁点 **导出**：得到 `{名称}-response-body.json` 或 `{名称}-response-body.txt`（仅响应体）；超大正文流式下载时为 `{名称}-response-raw-body.txt`。
7. 修改名称、URL、参数等会在适当时机自动保存；也可在侧栏删除 API 或整个分组。
